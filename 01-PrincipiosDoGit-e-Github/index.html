Princípios do Git e GitHub

Git é um sistema de controle de versão distribuído, gratuito e de código aberto, projetado para lidar com tudo, desde projetos pequenos a muito grandes, com rapidez e eficiência.

GitHub é uma plataforma de hospedagem de código-fonte com controle de versão usando o Git. Ele permite que você e outras pessoas trabalhem juntas em projetos de qualquer lugar.

Comandos básicos do Git

  1. git init: Inicializa um repositório Git em um diretório existente.
  2. git clone: Clona um repositório Git existente para o diretório local.
  3. git add: Adiciona arquivos ao índice.
  4. git commit: Registra alterações no repositório.
  5. git status: Exibe o status das alterações como arquivos não rastreados, arquivos modificados e arquivos prontos para commit.
  6. git log: Exibe o histórico de commits.
  7. git branch: Lista, cria ou exclui branches.
  8. git checkout: Alterna entre branches ou restaura arquivos.
  9. git merge: Mescla branches.
  10. git pull: Atualiza o repositório local com as alterações do repositório remoto.
  11. git push: Envia commits para o repositório remoto.
  12. git remote: Lista os repositórios remotos configurados.
  13. git reset: Desfaz alterações em um arquivo específico.
  14. git rm: Remove um arquivo de um diretório e o inclui no próximo commit.
  15. git remote add: Adiciona um repositório remoto com um nome específico.
  16. git fetch: Busca commits de um repositório remoto.
